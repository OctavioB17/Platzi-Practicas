Platzi Semantic HTML Practice

Semantic HTML refers to the use of HTML tags that convey the meaning of the content contained within them, rather than just the way it looks. This approach helps search engines, screen readers, and other tools understand the structure and importance of the content on a web page.

  1. Use of Proper Tags:
    • Use headings (< h1 >, < h2 >, etc.) to define sections and sub-sections.
    • Use < article >, < section >, < nav >, and < aside > to organize content.
    • Use < header > and < footer > for the top and bottom parts of content.
  2. Accessibility:
    • Use < main > to encapsulate the main content, helping screen readers navigate.
    • Ensure all images have alt text using the < img > element's alt attribute.
    • Use < label > for form controls and < fieldset > for grouping form elements.
  3. Forms and Inputs:
    • Use semantic tags for forms, such as < form >, < input >, < textarea >, and < button >.
    • Use < label > to associate labels with form controls for better accessibility.
  4. Lists and Navigation:
    • Use < ul > and < ol > for unordered and ordered lists, respectively.
    • Use < nav > for navigation links to help assistive technologies identify menus.
  5. Tables and Data:
    • Use < table >, < thead >, < tbody >, and < tfoot > to structure tabular data.
  6. Document Structure:
    • Use < title > to give the document a meaningful title.
    • Use < meta > tags to provide metadata about the document.
